말로 시키는 AI 업무 도우미 — 사내 모든 도구를 자연어로 제어하세요

1 Dexus란?

Jira, Confluence, Slack, Google Workspace... 매일 쓰는 도구를 하나의 대화창에서 모두 제어하는 AI 업무 도우미입니다.

기존 방식
  • Jira 열어서 이슈 검색
  • Confluence 열어서 문서 작성
  • 캘린더 따로, 메일 따로 확인
  • 키워드 정확하게 기억해야 검색 가능
  • 도구 간 복사-붙여넣기 반복
Dexus 방식
  • "오늘 내 Jira 이슈 뭐 있어?"
  • "이 내용으로 Confluence 문서 만들어줘"
  • "이번 주 캘린더 보여줘"
  • 자연어로 검색 — 키워드 몰라도 OK
  • 도구 간 자동 연결 (한 문장으로 3가지)

이렇게 대화하면 돼요

"오늘 내 Jira 이슈 뭐 있어?"
PROJ-123 [긴급] 버그 수정, PROJ-124 API 리팩토링... 총 4건이에요.
"이번 주 캘린더 보여줘"
월요일 10시 스프린트 플래닝, 화요일 2시 디자인 리뷰...
"이 내용으로 Confluence 페이지 만들어줘"
새 페이지를 만들었어요! 제목: "API 가이드 v2" 링크: ...

이것만 기억하세요!

  • Dexus는 Jira, Confluence, Slack, Google 등 사내 도구를 자연어로 제어하는 AI 도우미입니다
  • 키워드를 외울 필요 없이, 평소 말하듯 대화하면 됩니다
  • 한 문장으로 여러 도구를 조합해서 쓸 수 있습니다

Atlassian (Jira + Confluence)

OAuth 인증으로 연동

연동 방법

API Tokens 탭 → Atlassian → OAuth 로그인

할 수 있는 것

  • Jira 이슈 조회/생성/수정
  • Confluence 문서 검색/생성/수정
  • 스프린트 현황 조회

Google Workspace

OAuth 인증으로 연동

연동 방법

API Tokens 탭 → Google → OAuth 로그인

할 수 있는 것

  • Gmail 이메일 확인/초안 작성
  • Calendar 일정 조회/생성
  • Drive 파일 검색
  • Sheets 데이터 조회/편집

Slack

OAuth 인증으로 연동

연동 방법

API Tokens 탭 → Slack → OAuth 로그인

할 수 있는 것

  • 채널 메시지 조회/전송
  • 스레드 요약
  • 워크플로우 자동화 연동

Bitbucket

OAuth 인증으로 연동

연동 방법

API Tokens 탭 → Bitbucket → OAuth 로그인

할 수 있는 것

  • PR 목록 조회
  • PR 설명 작성
  • 브랜치 정보 확인
연결 테스트를 꼭 해주세요! 각 서비스 연동 후, 간단한 질문으로 연결이 잘 됐는지 확인해보세요. 예: "내 Jira 이슈 보여줘"
도구 활성화/비활성화 Settings > Tools에서 개별 도구를 켜고 끌 수 있습니다. LV1 도구는 필수, LV2 도구는 필요에 따라 선택하세요.

이것만 기억하세요!

  • API Tokens 탭에서 서비스별 OAuth 로그인으로 연동
  • Atlassian(Jira+Confluence)은 필수 연동
  • 연동 후 간단한 질문으로 연결 테스트 확인

5 핵심 연동 기능 활용

개발/PM팀 실전편 — 이런 상황에서 이렇게 쓰세요

5-1 Slack ↔ Jira/Confluence 양방향 연동

시나리오 1: Slack 스레드 → Jira 이슈 자동 생성

버그 논의가 길어졌을 때, 스레드를 Jira 이슈로 바로 전환

상황Slack에서 버그 논의 중
→
Dexus에게"이 Slack 스레드 내용으로 Jira 버그 이슈 만들어줘"
→
결과Jira 이슈 자동 생성 + 링크 공유
시나리오 2: Slack 스레드 → Confluence 페이지 생성

기술 논의나 의사결정 내용을 문서화

상황기술 논의가 결론에 도달
→
Dexus에게"이 스레드 내용 정리해서 Confluence 문서로 만들어줘"
→
결과정리된 Confluence 페이지 생성
시나리오 3: Jira/Confluence → Slack 답변

슬랙에서 스펙이나 이슈 관련 질문이 나왔을 때

상황Slack에서 "이 기능 스펙이 어떻게 되지?"
→
Dexus에게"@Dexus 이 기능 Confluence 스펙 문서 찾아서 요약해줘"
→
결과관련 문서 검색 + 요약 답변
시나리오 4: Jira 이슈 현황 → Slack 공유

스프린트 현황을 슬랙 채널에 공유

상황스프린트 현황 공유가 필요
→
Dexus에게"현재 스프린트 이슈 현황 정리해서 #dev-team에 공유해줘"
→
결과현황 요약 메시지 Slack 전송

5-2 Slack 워크플로우 자동화

핵심 포인트: Slack 워크플로우에서 Dexus를 호출하면, 각종 조회/리포트가 자동화됩니다.

시나리오 5: 모니터링 알림 → 자동 Jira 이슈 생성

워크플로우가 @Dexus 멘션하여 자동으로 이슈 생성

워크플로우모니터링 알림 감지
→
자동 실행@Dexus "이 알림 내용으로 Jira 이슈 만들어줘"
→
결과Jira 이슈 자동 생성
시나리오 6: 정기 리포트 자동화

매주 스프린트 현황을 자동으로 채널에 공유

워크플로우매주 월요일 오전 트리거
→
자동 실행@Dexus "이번 주 스프린트 현황 리포트 만들어줘"
→
결과정기 리포트 Slack 자동 게시
워크플로우 자동화를 하려면 사전 설정이 필요해요
  • Slack에서 /dexus grant-mention [봇이름]으로 워크플로우 봇을 등록해야 합니다
  • 등록은 채널 단위로 적용됩니다
  • Slack Reminder(/remind)는 호환되지 않습니다 — 반드시 워크플로우를 사용하세요

"먼저 현재 내용 보여줘" → 확인 후 → "이제 수정해줘"

"이 이미지 내용 요약해줘" / "이 디자인 피드백 줘"

팀 지식 쌓기
